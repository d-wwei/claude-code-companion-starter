# Claude Code 个人助理文件系统初始化提示词

## 使用方法

把 [`step1-init.md`](file:///Users/eli/Documents/CC飞书个人助手/step1-init.md) 中 ` ```text ``` ` 代码块里的内容发给 Claude Code 即可。

**只需要发一次。** 全局规则写入 `~/.claude/` 后永久生效——以后进入任何新项目，Claude Code 会自动：
1. 检查 `.assistant/` 是否存在
2. 不存在则自动创建项目文件结构
3. 检查 bootstrap 状态，需要时自动进入引导对话
4. 从文件系统中读取用户偏好和记忆，而不是重复提问

## 文件说明

| 文件 | 用途 |
|---|---|
| `step1-init.md` | **发给 Claude Code 的提示词**（~210 行） |
| `README.md` | 飞书个人助手完整方案与教程 |
| 本文件 | 提示词说明文档 |

## 提示词做了什么

### 全局层 `~/.claude/`
| 文件 | 作用 |
|---|---|
| `CLAUDE.md` | 入口文件，`@` 导入其他规则（不支持时自动合并写入） |
| `assistant-core.md` | 角色行为、读取顺序、冲突解决、快速回顾入口 |
| `bootstrap-rules.md` | 冷启动规则：分层提问、状态追踪、完成持久化 |
| `memory-policy.md` | 记忆写入/冲突/清理/审计规则 |
| `project-filesystem.md` | 项目文件结构定义、Git 安全处理 |

### 项目层 `.assistant/`
每个项目自动创建的目录结构，包含用户画像、风格偏好、工作流、记忆分层和模板系统。

## 包含的优化

1. `@` 导入兼容性检测 + 不支持时自动合并回退
2. 幂等性判断标准（3 行有效内容规则）
3. 写入后逐文件校验
4. daily 日志生命周期（7/14 天清理策略）
5. `last-session.md` 写入时机规则
6. 记忆冲突解决机制
7. Bootstrap 完成状态持久化（`status` 字段）
8. Bootstrap 问题优先级分层（必问/推荐/自然积累）
9. 用户审计权（查看/删除/导出/月度提醒）
10. `.gitignore` 自动处理
11. 模板系统灵活化（bootstrap 中询问自定义模板）
12. 快速回顾入口（"查看我的配置"）
13. 工作区确认（防止在非项目目录创建 `.assistant/`）
